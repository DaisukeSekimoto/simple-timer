# Simple Timer

日報に記載する作業時間を、1日単位で記録するシンプルなタイマーです。

カウントダウンとストップウォッチに対応しており、計測した作業を当日の履歴として保存できます。サーバーやアカウントは不要で、GitHub Pagesなどの静的サイトとして公開。

## 主な機能

- カウントダウン／ストップウォッチの切り替え
- 計測した作業を当日の日付で記録
- 日付を選択して過去の作業履歴を表示
- タイマーを使わなかった作業履歴を日付・タスク名・作業時間の手入力で追加
- 保存されている全履歴をTXTファイルとして一括出力
- 作業時間を「分」または「hh:mm:ss」で表示
- 最近使用したタスク名を最大10件表示
- タイマーの小窓表示
- Picture-in-Picture内でのタスク名入力・履歴表示・履歴追加
- 端末・ブラウザごとの履歴保存

## 使い方

1. 必要に応じて「タスク名を入力」を押し、作業名を入力します。
2. カウントダウンまたはストップウォッチを選択します。
3. 「開始」を押して計測を始めます。タスク名が未入力でも計測できます。
4. 作業が終わったら「次の作業に移る」を押します。
5. 計測結果が今日の履歴に保存され、追加完了の通知が数秒間表示されます。

「次の作業に移る」を押した時点でタスク名が未入力の場合は、タスク名の入力画面が表示されます。タスク名を決定すると、直前の計測結果がその名前で保存されます。

「リセット」を押すと、計測時間と入力中のタスク名が初期化されます。

画面右上の「履歴」から日付を選択すると、過去の記録を確認できます。履歴画面の初期日付は今日、時間表示の初期値は「分」です。分表示では秒数を四捨五入します。

履歴画面の「全履歴をTXT出力」を押すと、保存されているすべての日付の記録を1つのTXTファイルとしてダウンロードできます。画面上で選択している表示形式にかかわらず、作業名、分、`hh:mm:ss`がそれぞれ同じ列位置にそろった表形式で出力されます。分の値は秒数を四捨五入します。

履歴内の記録をクリックすると確認画面が表示され、選択した記録を削除できます。

タイマーの開始を忘れた場合は、履歴画面の「作業履歴を追加」を押してください。日付、タスク名、作業時間を入力すると、通常のタイマー記録と同じ履歴に追加されます。

## 履歴の保存について

履歴はサーバーには送信されず、使用中のブラウザのストレージに保存されます。そのため、同じ公開URLを同じ端末・同じブラウザで開いた場合に限り、以前の履歴を参照できます。

**ブラウザのキャッシュ、Cookie、サイトデータ、閲覧データなどを削除すると、保存されている履歴も消える可能性があります。履歴を残しておきたい場合は、このサイトのデータを削除しないでください。**

次の場合も履歴を参照できなくなることがあります。

- 別の端末や別のブラウザからアクセスした場合
- シークレットモード／プライベートブラウズを使用した場合
- GitHub PagesのURLやドメインを変更した場合
- ブラウザを初期化またはアンインストールした場合

## ファイル構成

- `index.html`：画面構成
- `styles.css`：デザインとレイアウト
- `app.js`：タイマー、履歴、ブラウザ保存の処理

## 動作環境

最新版のGoogle Chrome、Microsoft Edge、Safari、Firefoxなど、`localStorage`と`dialog`要素に対応したブラウザを使用してください。
